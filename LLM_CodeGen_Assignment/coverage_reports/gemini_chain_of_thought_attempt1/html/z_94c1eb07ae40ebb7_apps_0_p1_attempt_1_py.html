8 ''' 

9 You are given N items, each with a weight w[i] and value v[i]. 

10 You have a knapsack with capacity W. 

11 However, there's a twist: you can take at most K items of each type. 

12  

13 Find the maximum value you can achieve. 

14  

15 Parameters: 

16 - n: number of item types 

17 - W: knapsack capacity  

18 - k: max items per type 

19 - items: list of (weight, value) tuples 

20  

21 Example: 

22 n=3, W=10, k=2 

23 items=[(2,3), (3,4), (4,5)] 

24  

25 You can take at most 2 of each item type. 

26 Optimal: Take 2 items of type 1 (weight=4, value=6) and 1 item of type 2 (weight=3, value=4) 

27 Total: weight=7, value=10 

28  

29 Return the maximum value achievable. 

30 ''' 

31 

32 # This problem can be modeled as a variation of the unbounded knapsack problem, 

33 # but with a limit on the number of items of each type. 

34 # We can transform this into a 0/1 knapsack problem by "unrolling" the items. 

35 # For each item type (weight_i, value_i), we can take 0, 1, 2, ..., k of them. 

36 # This means we can consider k copies of each item type, each with the same weight and value. 

37 # However, this would lead to a very large number of items if k is large. 

38 

39 # A more efficient approach is to use dynamic programming. 

40 # Let dp[i][j] be the maximum value that can be obtained using the first i item types 

41 # with a knapsack capacity of j. 

42 

43 # The state transition would involve considering how many items of the i-th type we take. 

44 # dp[i][j] = max(dp[i-1][j - count * weight_i] + count * value_i) for count from 0 to k 

45 # such that j - count * weight_i >= 0. 

46 

47 # We can optimize space by using only one row for DP. 

48 # Let dp[j] be the maximum value for a knapsack of capacity j. 

49 # When considering item type i (weight_i, value_i), we iterate through the knapsack 

50 # capacities from W down to 0. 

51 # For each capacity j, we consider taking 'count' items of type i. 

52 # dp[j] = max(dp[j], dp[j - count * weight_i] + count * value_i) for count from 1 to k. 

53 

54 # Initialize DP table. dp[j] will store the maximum value for capacity j. 

55 dp = [0] * (W + 1) 

56 

57 # Iterate through each item type 

58 for weight_i, value_i in items: 

59 # Iterate through knapsack capacities from W down to 0 

60 # We iterate downwards to ensure that when we update dp[j], 

61 # we are using values from the previous item type's calculations, 

62 # effectively simulating the dp[i-1] state. 

63 for j in range(W, -1, -1): 

64 # Consider taking 'count' items of the current type 

65 for count in range(1, k + 1): 

66 current_weight = count * weight_i 

67 current_value = count * value_i 

68 

69 # If taking 'count' items of the current type does not exceed capacity j 

70 if j >= current_weight: 

71 # Update dp[j] by considering the possibility of taking 'count' items 

72 # of the current type. The value would be the value obtained from 

73 # the remaining capacity (j - current_weight) plus the value of 

74 # the 'count' items. 

75 dp[j] = max(dp[j], dp[j - current_weight] + current_value) 

76 else: 

77 # If taking 'count' items exceeds capacity j, we can't take more 

78 # of this item type for this capacity, so break the inner loop.
