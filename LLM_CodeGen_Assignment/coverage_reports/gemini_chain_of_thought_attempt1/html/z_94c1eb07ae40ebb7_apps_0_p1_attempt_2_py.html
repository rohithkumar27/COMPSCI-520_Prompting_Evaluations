7def solve_knapsack_variant(n: int, W: int, k: int, items: List[tuple[int, int]]) -> int: 

8 ''' 

9 You are given N items, each with a weight w[i] and value v[i]. 

10 You have a knapsack with capacity W. 

11 However, there's a twist: you can take at most K items of each type. 

12  

13 Find the maximum value you can achieve. 

14  

15 Parameters: 

16 - n: number of item types 

17 - W: knapsack capacity  

18 - k: max items per type 

19 - items: list of (weight, value) tuples 

20  

21 Example: 

22 n=3, W=10, k=2 

23 items=[(2,3), (3,4), (4,5)] 

24  

25 You can take at most 2 of each item type. 

26 Optimal: Take 2 items of type 1 (weight=4, value=6) and 1 item of type 2 (weight=3, value=4) 

27 Total: weight=7, value=10 

28  

29 Return the maximum value achievable. 

30 ''' 

31 

32 # This problem can be modeled as a bounded knapsack problem. 

33 # Since we can take at most k items of each type, we can effectively 

34 # "unroll" each item type into k individual items. 

35 # For example, if we have item type (w_i, v_i) and k=2, we can think of 

36 # it as two separate items: (w_i, v_i) and (w_i, v_i). 

37 # However, this can lead to a large number of items if k is large. 

38 # A more efficient approach for bounded knapsack is to use dynamic programming. 

39 

40 # dp[i][j] will store the maximum value that can be obtained with a knapsack 

41 # of capacity j using the first i item types. 

42 # However, since we can take multiple items of the same type, a standard 

43 # 0/1 knapsack DP won't directly work. 

44 

45 # We can transform this into a 0/1 knapsack problem by considering each 

46 # item type and the number of copies we can take. 

47 # For each item type (weight, value), we can take 0, 1, 2, ..., k copies. 

48 # This can be further optimized by using binary splitting for the number of items. 

49 # For example, if k=7, we can represent 7 as 1 + 2 + 4. So, instead of 

50 # considering 7 copies of an item, we can consider items with weights 

51 # 1*w, 2*w, 4*w and corresponding values 1*v, 2*v, 4*v. This reduces the 

52 # number of effective items to O(N log K). 

53 

54 # Let's create a new list of "effective" items. 

55 effective_items = [] 

56 for weight, value in items: 

57 count = 1 

58 while k > 0: 

59 take = min(count, k) 

60 effective_items.append((weight * take, value * take)) 

61 k -= take 

62 count *= 2 

63 k = k # reset k for the next item type 

64 

65 # Now, we have a list of items where each item can be taken at most once. 

66 # This is a standard 0/1 knapsack problem. 

67 # dp[j] will store the maximum value for a knapsack of capacity j. 

68 dp = [0] * (W + 1) 

69 

70 for item_weight, item_value in effective_items: 

71 # Iterate from W down to item_weight to avoid using an item multiple times 

72 # within the same capacity calculation. 

73 for current_W in range(W, item_weight - 1, -1): 

74 dp[current_W] = max(dp[current_W], dp[current_W - item_weight] + item_value) 

75 

76 return dp[W]
